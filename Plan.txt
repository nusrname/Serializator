---

# 🔴 Главная ошибка, которую важно избежать

Сравнивать «JSON vs бинарь» напрямую на текущем коде — некорректно, потому что

 у тебя сейчас разные пайплайны
 разные аллокации
 разный объём данных (например, JSON pretty print)

👉 результат будет шумным и неубедительным

---

# ✅ ПРАВИЛЬНАЯ СХЕМА ЭКСПЕРИМЕНТА

## 1. Ввести интерфейс сериализации

```csharp
public interface ISaveSerializer
{
    byte[] Serialize(GameSaveData data);
    GameSaveData Deserialize(byte[] data);
}
```

---

## 2. Реализации

### ✔ JSON (baseline)

```csharp
public class JsonSerializerImpl  ISaveSerializer
{
    public byte[] Serialize(GameSaveData data)
    {
        string json = JsonUtility.ToJson(data, false);  ВАЖНО без pretty
        return System.Text.Encoding.UTF8.GetBytes(json);
    }

    public GameSaveData Deserialize(byte[] data)
    {
        string json = System.Text.Encoding.UTF8.GetString(data);
        return JsonUtility.FromJsonGameSaveData(json);
    }
}
```

---

### ✔ Твоя C++ DLL

```csharp
public class CustomBinarySerializer  ISaveSerializer
{
    public byte[] Serialize(GameSaveData data)
    {
         1. преобразовать GameSaveData → примитивы  DTO
         2. передать в DLL
         3. получить byte[]
        return NativeSerializer.Serialize(data);
    }

    public GameSaveData Deserialize(byte[] data)
    {
        return NativeSerializer.Deserialize(data);
    }
}
```

# 🧪 4. КАК ПРОВОДИТЬ ТЕСТ

## ✔ Сделай отдельный класс

```csharp
public static class SaveBenchmark
{
    public static void Run(ISaveSerializer serializer, string name)
    {
        var data = SaveSystem.SaveData;

        var sw = Stopwatch.StartNew();
        var bytes = serializer.Serialize(data);
        sw.Stop();

        long serializeTime = sw.ElapsedMilliseconds;
        int size = bytes.Length;

        sw.Restart();
        var loaded = serializer.Deserialize(bytes);
        sw.Stop();

        long deserializeTime = sw.ElapsedMilliseconds;

        Debug.Log($[{name}] Size {size} bytes  Save {serializeTime} ms  Load {deserializeTime} ms);
    }
}
```

---

## ✔ Запуск

```csharp
SaveBenchmark.Run(new JsonSerializerImpl(), JSON);
SaveBenchmark.Run(new CustomBinarySerializer(), Custom Binary);
```

# 🧠 ВАЖНЫЙ ВЫВОД (для курсовой)

Ты должен показать

1. Компромисс

    JSON → читаемость
    Binary → производительность

2. Практическую применимость

    выигрыш при росте данных
    влияние на загрузку игры

3. Ограничения

    сложность поддержки бинарного формата
    зависимость от версии
